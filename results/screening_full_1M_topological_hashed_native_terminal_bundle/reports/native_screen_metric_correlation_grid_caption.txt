Figure caption
Ligand-level correlations between four screening-stage metrics and native weighted 3D coverage for the top 4,997 native-scored ligands from the `full_1M_topological_hashed` run. Each panel shows an ordinary least-squares trend line with a 95% bootstrap confidence band based on 2,000 resamples.

Panel A (pair-hash overlap): Pearson r = 0.275 (95% bootstrap CI 0.248 to 0.303), Spearman rho = 0.250 (95% bootstrap CI 0.224 to 0.279), OLS slope = 0.218, R^2 = 0.076.
Panel B (pair-hash recall): Pearson r = 0.292 (95% bootstrap CI 0.264 to 0.319), Spearman rho = 0.260 (95% bootstrap CI 0.234 to 0.287), OLS slope = 0.299, R^2 = 0.085.
Panel C (hotspot-compatible matches): Pearson r = 0.306 (95% bootstrap CI 0.272 to 0.338), Spearman rho = 0.221 (95% bootstrap CI 0.192 to 0.251), OLS slope = 1.051, R^2 = 0.094.
Panel D (cascade score): Pearson r = 0.113 (95% bootstrap CI 0.085 to 0.140), Spearman rho = 0.106 (95% bootstrap CI 0.079 to 0.135), OLS slope = 0.106, R^2 = 0.013.

Metric definitions:
0. `native_weighted_coverage_pct` is the final native-branch 3D mimicry score on the y-axis of every panel. It is computed as `100 * matched_reference_weight / total_reference_feature_weight`, where `matched_reference_weight` is the summed weight of the retained native GLP-1 reference features matched by the ligand's best aligned conformer and `total_reference_feature_weight` is the summed weight of all retained reference features in the active peptide model.
1. `pair_hash_overlap_pct` is the stage-2 typed-pair overlap score reported as a percentage. The code first builds weighted pharmacophore pair buckets from the selected receptor-side pair query and weighted ligand pair buckets from the ligand's typed atom features, both under the active `precision_5bin` distance-bucket scheme. It then computes weighted pair-hash recall and weighted pair-hash precision; because `precision_5bin` is precision-aware, the stored overlap is `100 * 2 * precision * recall / (precision + recall)`.
2. `pair_hash_recall_pct` is `100 * matched_query_weight / total_query_weight`, where `matched_query_weight` is the summed weight of receptor-side pair buckets that can be supported by compatible ligand pair buckets after applying the bucket-count matching fraction and any family/bucket credit.
3. `hotspot_compatible_matches` is a count, not a percentage. In stage 1, each receptor-derived hotspot is first tested for an exact family match in the ligand feature inventory. If exact matching fails but a compatible feature family is available, that hotspot contributes one compatible match. These compatible matches receive half credit in the hotspot-weighted score via `COMPATIBLE_MATCH_CREDIT = 0.5`.
4. `cascade_score_pct` is the prescreen cascade score stored before the expensive 3D rerank. It is computed as `100 * (0.4 * hotspot_weighted_fraction + 0.6 * pair_hash_overlap)`, where `hotspot_weighted_fraction` is the receptor-hotspot weighted score after exact and half-credit compatible matches, and `pair_hash_overlap` is the stage-2 overlap fraction described above.

All four screen-derived metrics show positive, statistically detectable but modest associations with native weighted 3D coverage. The effect sizes remain limited, which supports treating the topological cascade as an enrichment-and-triage system rather than as a direct surrogate for the final native pharmacophore score.
The strongest linear association is observed for hotspot-compatible matches (Pearson r = 0.306; R^2 = 0.094), while the strongest rank-based association is observed for pair-hash recall (Spearman rho = 0.260). The weakest of the four is cascade score (Pearson r = 0.113; R^2 = 0.013), indicating that the blended cascade score is useful for shortlist triage but is not a close proxy for downstream native 3D mimicry within the native-scored cohort.
Taken together, the figure supports the architecture of the Topological-to-3D Native Pharmacophore Cascade: earlier hotspot and pair-hash filters retain measurable information about native mimicry, but the modest correlations justify keeping the explicit downstream 3D/native rerank rather than collapsing the workflow into a single topological score.
Hotspot-compatible-match points are plotted with slight horizontal jitter to reveal density, but all reported statistics were computed from the original discrete counts.
